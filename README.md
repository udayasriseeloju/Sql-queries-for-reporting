# Sql-queries-for-reporting
#  Sales Report SQL Project
## Project Overview
**Database:** `SalesDB`  
**Tool Used:** MySQL Workbench  
**Main Deliverables:**
- SQL Query File (`sales_reports_full.sql`)
- Query Output Screenshots
- Summary Report



## Features
- Create and populate a relational sales database  
- Calculate **Daily Sales Totals**  
- Compute **Average Transaction Values** (overall and per customer)  
- Identify **Top Products** by revenue and quantity  
- View **Monthly Revenue Trends**  
- Find **Top Customers** by spending  



##  Files Included
`sales_reports_full.sql` – Complete SQL script to create database, tables, sample data, and reports  
  `README.md` – Project documentation  
  `summary_report.pdf` (optional) – Descriptive summary of results  



##  How to Run
1. Open **MySQL Workbench**.  
2. Create a new SQL tab and paste the contents of `sales_reports_full.sql`.  
3. Execute the script 
4. Run each **REPORT** block separately to view results.  
5. Capture screenshots or export result sets for documentation.



**Reports Generated**
1. **Daily Totals** – Total revenue, orders, and items sold per day  
2. **Average Transactions** – Average order values (overall and by customer)  
3. **Top Products** – Best-performing products by sales and quantity  
4. **Monthly Revenue Trend** – Revenue grouped by month  
5. **Top Customers** – Customers with highest spending
